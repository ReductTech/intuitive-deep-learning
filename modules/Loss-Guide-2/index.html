Sigmoid 如何把原始分数变成概率？

σ(z) = 1 1 + e−z
z ∈ ℝ

Sigmoid 会把任意实数映射到 0～1 之间，因此它的输出可以用来表示“明天下雨”的概率。

观察 Sigmoid 的变化

Sigmoid 可以接收任意实数。原始分数越大，输出越接近 1；原始分数越小，输出越接近 0。对于有限的原始分数，输出只会无限接近 0 或 1，不会真正到达两个端点。

第二步 · 计算二分类损失

预测概率有了，怎样衡量预测得好不好？

假设第二天确实下雨，真实标签为 1。请设计一个损失函数，让预测概率越接近 1，损失越小。

你的损失函数
负对数损失
L(p) = −log(p)
L′(p) = −1/p
把两种真实情况写成一个公式 当真实标签 y=1 时，只计算实际下雨对应的损失；当 y=0 时，只计算实际不下雨对应的损失。把两种情况写在一起，就得到二分类交叉熵损失 BCE。
情况一：真实值为 0，也就是实际没下雨

正确类别是“不下雨”，它的概率是 1 − p。

L = −log(1 − p)
情况二：真实值为 1，也就是实际下雨

正确类别是“下雨”，请填入它的预测概率。

L = −log()

先写成分段函数
L(p, y) = { −log(1 − p)真实值 y = 0 时 −log(p)真实值 y = 1 时
用真实标签 y 选择对应的损失
L = −[y log(p) + (1 − y) log(1 − p)]

为什么多个类别不能分别使用 Sigmoid？

以天气预测为例：晴、阴、雨只能选一个，三个类别的概率应合计 100%。

每个类别独立使用 Sigmoid
pi = σ(zi)

三个 Sigmoid 互不约束，总和可能超过 100%。

概率总和 0.0% 三选一应为 100%

Softmax 如何让多个类别共同竞争？

Softmax 会把三个原始分数（logit）一起归一化，得到总和为 100% 的类别概率。

pk = ezk ez晴 + ez阴 + ez雨

共同的分母保证三个概率之和为 1。

多分类预测错了，损失怎样计算？

用多分类交叉熵计算预测损失

LCE = −Σ yk log(pk)

如果任务是单标签分类任务（一个样本只属于一个类别），那么交叉熵公式可以进一步简化为

LCE = −log(p下雨)
真实类别概率越高，CE 越小
