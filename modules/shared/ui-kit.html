提示框

颜色和播放方式是两个独立维度。橙、蓝、绿、红都可以直接显示，也都可以逐字显示；不要用颜色判断是否自动播放。

你的任务 拖动绿色预测值，让它与红色真实值重合，把 Loss 缩小到 0。
思考提示 先比较预测值和真实值，再判断应该向左还是向右移动。
观察结果 预测值越接近真实值，当前样本产生的损失越小。
逐步解释 预测值每靠近真实值一步，损失就会随距离一起减小。
完成提示 预测值已经与真实值重合，本轮损失降到了 0。
正确反馈 方向判断正确，继续缩小距离就能进一步降低损失。
错误提示 预测值正在远离真实值，当前操作会让损失继续增大。
风险提醒 学习率过大可能越过最低点，导致训练过程来回震荡。

紧凑提示条

用于紧跟图表、模型或控制区显示一句当前状态。它比提示框更轻，不承载长解释、任务说明或答题反馈。

操作提醒： 继续调整参数，比较曲线改变前后的形状。
阶段完成： 当前结果已经满足目标，可以进入下一步。
需要调整： 当前参数使输出偏离目标，请检查输入范围。

代码运行块

代码主体只读并使用语法颜色区分。需要学习者完成的位置只能嵌入单行填空，同时保留语言、运行状态、启动、停止、运行时间和请求帮助。

Python 就绪 运行时间0.0 s 
import torch
 
prediction = torch.tensor([1.6])
target = torch.tensor([7.0])
loss = torch.(target - prediction)
 
print(f"L2 loss: {loss.item():.2f}")
这里需要填写 PyTorch 中执行平方运算的函数名。

按钮

按钮样式由操作语义决定，不按模块单独设计。同一区域通常只保留一个主操作。

默认按钮

用于次要操作，例如切换样本、重置局部设置或返回上一步，不应抢过主任务的视觉优先级。

.edu-btn

主操作按钮

用于当前阶段最重要的动作，例如开始训练、提交答案或进入下一步。同一区域通常只出现一个。

.edu-btn .edu-btn--primary

警告按钮

用于会明显改变当前实验结果、但仍然可以恢复的操作。点击前需要让学习者预期到变化。

.edu-btn .edu-btn--warn

危险按钮

仅用于删除、清空或不可直接撤销的操作。普通错误反馈不能使用危险按钮代替。

.edu-btn .edu-btn--danger

提示点击按钮

用于下一步已经出现、但学习者可能没有注意到的情况。初始发光，首次鼠标移入、键盘聚焦或点击后永久停止提示。

.dl-button-hint · [data-dl-button-hint]

询问按钮

用于就近解释概念或参数。鼠标悬浮、键盘聚焦或触屏点击时显示解释框；解释框进入浏览器最高显示层，不受卡片裁剪。

.edu-btn--explain · [data-dl-explain]

等待按钮

用于请求已经提交、正在等待结果的短暂状态。按钮不可重复点击，旋转边缘表示处理仍在继续；普通的前置条件禁用状态不使用动画。

.edu-btn .is-loading · disabled · aria-busy="true"

禁用按钮

用于前置条件尚未满足、当前操作不可执行的状态。必须在附近说明解锁条件；禁用期间保持静态，不显示等待动画。

.edu-btn .edu-btn--primary · disabled

参数与选项控件

根据数据类型选择控件：单项选择用下拉或单选，多项选择用复选框，二元设置用开关，数值范围用滑杆。

提示输入框

用于输入任务已经出现、但学习者可能没有注意到的情况。初始持续闪烁，首次鼠标移入、键盘聚焦、点击或输入后永久停止提示。

.edu-input · [data-dl-input-hint] · DLModuleUI.bindInputHints()

下拉列表

.edu-selectbox · DLModuleUI.bindSelectbox()

复选框：普通与候选

普通样式用于设置列表；带边框样式用于需要被看作独立候选项的特征、标签或筛选条件。

.edu-check · .edu-check--option
激活函数

单选组

.edu-radio-group .edu-radio

滑块开关

用于立即生效且持续保持的二元设置，例如显示或隐藏辅助信息。

.edu-switch

固定样式滑杆

适合已有默认值的参数。页面打开时直接显示当前值，拖动只更新数值，不出现待操作提示。

提示操作滑杆

[data-dl-range].is-unset · .edu-control.is-unset
