1. UI Kit 是唯一设计基准
修改模块前，AI 必须先完整查看：
modules/shared/ui-kit.html
不能只搜索类名，也不能只复制视觉样式。必须同时检查：
完整 HTML 结构
交互绑定方式
配套 CSS
JavaScript API
UI Kit 附近的注释和 AGENT IMPLEMENTATION NOTE
组件是否包含不可省略的子元素
例如“推荐资源”不是只有视频列表，还固定包括：
标题
说明
视频列表
返回课程目录
学习下一个

2. 优先复用，禁止仿制
UI Kit 已有组件时，必须直接使用原组件/API，不能自己画一个“差不多的”。
典型要求：
训练曲线：DLPlot.mountTrainingHistory()
提示按钮：.dl-button-hint + [data-dl-button-hint]
提示滑杆：[data-dl-range].is-unset + .edu-control.is-unset
提示输入框：.edu-input + [data-dl-input-hint] + bindInputHints()
问题组件：使用 DLModuleUI.mountQuestion()
推荐视频：使用 DLModuleUI.renderRelatedVideos()
推荐资源：迁移完整 edu-content-block，包括导航按钮
“使用 UI Kit”指结构、行为、状态变化和视觉全部复用，不只是加一个 class。

3. 页面布局必须紧凑
教学页面不能留下没有信息价值的大空白。
默认原则：
优先横向并排相关内容
较小的信息块不要独占整行
双栏内容结束后，后续单栏内容应明确占满一行
同一行的卡片尽量等高
小卡内部内容允许上下堆叠，节省横向空间
删除为了滚动效果而制造的大面积空白
被用户划掉或标 X 的区域应完整删除，包括 HTML、CSS 和脚本引用
4. 教学流程要连续、即时
完成一道题后，不要要求用户继续下拉寻找下一步。
默认行为：
当前题完成后立即隐藏
下一题直接弹出
下一阶段内容直接出现或使用指定的 UI Kit 流程模式
已完成的选择题不长期占据页面
简答题完成后，再显示对应操作按钮
操作按钮应放在接下来实际操作的位置附近
只有用户明确要求“下拉提示”时，才使用 UI Kit 模式 2。
5. 提示组件必须具备真实状态变化
提示效果不能只是静态闪烁。
按钮提示：
初始闪烁
首次鼠标移入、键盘聚焦或点击后停止
使用 [data-dl-button-hint] 的自动绑定机制
不要用程序自动 focus() 导致提示刚出现就消失
滑杆提示：
初始不显示真实数值
初始呈提示闪烁状态
用户首次亲自拖动后解除 is-unset
随后显示真实值并进入固定样式
输入框提示也必须使用 UI Kit 的自动绑定逻辑。
6. 动态数据必须可视化
涉及训练过程时，优先让变化“看得见”。
例如：
每点击“更新一次”或“训练一次”，曲线新增一个点
震荡必须通过上下变化的数据点明显表现
Loss 下降使用真实训练历史动态更新
不要用静态文字或自制简易 SVG 代替 UI Kit 训练指标曲线
只要求 Loss 时，隐藏 Accuracy、图例和多余读数
不额外显示“第 5 次 · Loss 4.00”之类重复徽标，除非用户明确要求
7. 每个信息只出现一次
避免重复表达同一状态。
例如当前学习率：
如果已放进“训练操作”卡，就删除下方独立学习率区域
不再保留斜线示意图、重复状态条或说明横栏
Loss 已由曲线表示时，中间卡只保留标题和曲线
不再同时显示公式、预测值、当前 Loss 和另一套小曲线
8. 三栏训练区域的默认结构
最终训练区域优先采用：
左：输入真实值 GT
中：标题为 Loss 的 UI Kit 训练指标曲线
右：训练操作、当前学习率、训练按钮
下方直接接网络示意图，不插入额外状态横条或大面积留白。
9. 推荐资源必须完整迁移
推荐资源固定包含：
edu-content-block
edu-content-head
标题“推荐资源”
说明“继续观看与当前主题直接相关的视频，巩固刚才完成的学习内容。”
推荐视频列表
edu-resource-actions
左侧“返回课程目录”
右侧主按钮“学习下一个”
“学习下一个”的地址根据真实课程顺序设置，不能机械复制 UI Kit 示例地址。
10. AI 的执行检查清单
每次修改前：
阅读对应 UI Kit 完整区块。
阅读区块附近的注释。
确认组件所有固定子元素。
检查现有模块状态与用户标注。
每次修改后：
删除被替换组件的旧 HTML。
删除旧 CSS。
删除旧 JavaScript 引用。
检查重复 ID。
检查失效 DOM 引用。
运行 JavaScript 语法检查。
运行 git diff --check。
能打开页面时实际验证桌面和移动布局。
不能浏览器验证时明确说明，不能假装验证完成。
一句话概括：
用户要的是严格复用 UI Kit 的完整组件和交互规则，教学流程即时连续，布局紧凑无冗余，数据变化清晰可见；不能只模仿外观、漏掉组件组成，或留下重复信息和无意义空白。
